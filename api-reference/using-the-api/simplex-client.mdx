The Simplex client is the main entry point for the SDK. It combines AI capabilities with browser automation through Playwright.
We have broken the functions into [high level](#high-level-functions) and [low level](#low-level-functions) functions.
## Constructor

```python
client = Simplex(
    api_key: str,
    browser: Optional[Browser] = None
)
```

<ParamField path="api_key" type="string" required>
  Your API key for authentication
</ParamField>

<ParamField path="browser" type="playwright.sync_api.Browser">
  Playwright browser object. If not provided, a new headless browser instance will be created.
</ParamField>


## High-Level Functions
These functions provide an intuitive interface for automated workflows. They handle complex operations like taking screenshots, converting natural language to actions, and executing those actions.

### goto()
```python
def goto(self, url: str) -> None
```

<ParamField path="url" type="string" required>
  The URL to navigate to
</ParamField>

```python
client.goto("https://www.example.com")
```

### do()
```python
def do(self, step_description: str, annotate: bool = False) -> None
```

<ParamField path="step_description" type="string" required>
  Natural language description of what should be done (e.g. "Click the login button")
</ParamField>

<ParamField path="annotate" type="boolean" default="false">
  If true, visualizes the actions being performed
</ParamField>

```python
client.do("Click the login button")
client.do("Type 'hello@example.com' into the email field")
```

## Low-Level Functions
These atomic functions provide fine-grained control over browser interactions. They're useful for debugging or integrating with custom automation agents.

### find_element()
```python
def find_element(
    self,
    element_description: str,
    state: Optional[Image.Image] = None,
    annotate: bool = False
) -> List[int]
```

<ParamField path="element_description" type="string" required>
  Natural language description of the element to find
</ParamField>

<ParamField path="state" type="Image.Image">
  Screenshot of the current screen. If not provided, a new screenshot will be taken.
</ParamField>

<ParamField path="annotate" type="boolean" default="false">
  If true, adds a visual red border overlay around the found element
</ParamField>

```python
# Find a login button
coords = client.find_element("login button")
# Find with annotation
coords = client.find_element("submit button", annotate=True)
```

### step_to_action()
```python
def step_to_action(
    self,
    step_description: str,
    state: Optional[Image.Image] = None
) -> List[List[str]]
```

<ParamField path="step_description" type="string" required>
  Natural language description of the step to convert
</ParamField>

<ParamField path="state" type="Image.Image">
  Screenshot of the current screen. If not provided, a new screenshot will be taken.
</ParamField>

### execute_action()
```python
def execute_action(
    self,
    action: List[List[str]],
    state: Optional[Image.Image] = None,
    annotate: bool = False
) -> None
```

<ParamField path="action" type="List[List[str]]" required>
  A list containing action type and description pairs. Supported types:
  - `["CLICK", "element description"]` - Clicks on the described element
  - `["HOVER", "element description"]` - Hovers over the described element
  - `["TYPE", "text to type"]` - Types the specified text
  - `["SCROLL", "scroll amount"]` - Scrolls by the specified amount
  - `["WAIT", "milliseconds"]` - Waits for the specified duration
</ParamField>

<ParamField path="state" type="Image.Image">
  Screenshot of the current screen. If not provided, a new screenshot will be taken.
</ParamField>

<ParamField path="annotate" type="boolean" default="false">
  If true, visualizes the actions being performed
</ParamField>

```python
client.execute_action(["CLICK", "login button"])
client.execute_action(["TYPE", "hello@example.com"])
client.execute_action(["WAIT", "1000"])  # Wait for 1 second
```
