The Simplex client is the main entry point for the SDK. It combines AI capabilities with browser automation through its integrated BrowserActor.

## Constructor

```python
client = Simplex(
    api_key: str,
    driver: Optional[Page] = None
)
```

### Parameters

<ParamField path="api_key" type="string" required>
  Your API key for authentication
</ParamField>

<ParamField path="driver" type="playwright.sync_api.Page">
  Playwright page object. If not provided, a new headless browser instance will be created.
</ParamField>

## Functions

### Atomic Operations

The following functions are atomic operations that form the building blocks of more complex automation:

### find_element

```python
def find_element(
    self,
    element_description: str,
    state: Optional[Image.Image] = None,
) -> List[int]
```

Finds an element in a screenshot based on a natural language description.

#### Parameters

<ParamField path="element_description" type="string" required>
  Natural language description of the element to find
</ParamField>

<ParamField path="state" type="Image.Image">
  Screenshot of the current screen. If not provided, a new screenshot will be taken automatically.
</ParamField>

#### Returns
List of 4 integers representing the bounding box [x1, y1, x2, y2]

### step_to_action

```python
def step_to_action(
    self,
    step_description: str,
    state: Optional[Image.Image] = None
) -> List[List[str]]
```

Converts a natural language step description into a list of actions.

#### Parameters

<ParamField path="step_description" type="string" required>
  Natural language description of the step to convert
</ParamField>

<ParamField path="state" type="Image.Image">
  Screenshot of the current screen. If not provided, a new screenshot will be taken automatically.
</ParamField>

#### Returns
List of action pairs, where each pair is a list of two strings representing the action type and target

### Composite Operations

The following functions combine atomic operations to perform higher-level tasks:

### do

```python
def do(
    self,
    step_description: str
) -> None
```

Executes a natural language step by converting it to actions and performing them in sequence. This function combines `step_to_action` with action execution.

#### Parameters

<ParamField path="step_description" type="string" required>
  Natural language description of what should be done (e.g. "Click the login button")
</ParamField>

#### Implementation Details
1. Takes a screenshot of the current page
2. Uses `step_to_action` to convert the description into concrete actions
3. Executes each action in sequence

#### Example

```python
client.do("Click the login button")
client.do("Type 'hello@example.com' into the email field")
```

### execute_action

```python
def execute_action(
    self,
    action: List[List[str]]
) -> None
```

Executes a specific browser action using the Playwright driver. This function handles various types of interactions like clicking, typing, and scrolling.

#### Parameters

<ParamField path="action" type="List[List[str]]" required>
  A list containing action type and description pairs. Supported action types:
  - `["CLICK", "element description"]` - Clicks on the described element
  - `["HOVER", "element description"]` - Hovers over the described element
  - `["TYPE", "text to type"]` - Types the specified text
  - `["SCROLL", "scroll amount"]` - Scrolls by the specified amount
  - `["WAIT", "milliseconds"]` - Waits for the specified duration
</ParamField>

#### Implementation Details
1. Takes a screenshot of the current state
2. Based on the action type:
   - For CLICK/HOVER: Finds the element and interacts with its center
   - For TYPE: Inputs the specified text
   - For SCROLL: Moves the page vertically
   - For WAIT: Pauses execution

#### Example

```python
client.execute_action(["CLICK", "login button"])
client.execute_action(["TYPE", "hello@example.com"])
client.execute_action(["WAIT", "1000"])  # Wait for 1 second
```
